## 1. התקנת LocalWP

**מה זה LocalWP?**
LocalWP (בעבר "Local by Flywheel") הוא תוכנה חינמית שמתקינה WordPress על המחשב שלכם, בלי שרת חיצוני. מושלם לפיתוח ובדיקות.

### להורדה:
1. גשו ל-[localwp.com](https://localwp.com) ולחצו **Free Download**
2. בחרו את מערכת ההפעלה שלכם (Windows / macOS)
3. הורידו, פתחו את הקובץ והתקינו — כמו כל תוכנה רגילה
4. הפעילו את LocalWP לאחר ההתקנה

> **שים לב:** אין צורך להתחבר — ניתן לדלג על ה-sign up ולהמשיך ישירות.

## 2. יצירת אתר WordPress מקומי

1. פתחו את **LocalWP**
2. לחצו על **+** (כפתור הוספת אתר) בפינה השמאלית התחתונה
3. בחרו **Create a new site**
4. תנו שם לאתר, לדוגמה: `sunpepe`
5. לחצו **Continue**
6. תחת **Choose your environment**, בחרו **Preferred** ולחצו **Continue**
7. הגדירו שם משתמש וסיסמה לאדמין (כלשהם — זה מקומי), לחצו **Add Site**
8. המתינו עד שהאתר יהיה מוכן (כ-30 שניות)
9. לחצו **WP Admin** כדי לפתוח את לוח הבקרה של WordPress

## 3. העתקת הפלאגין לתיקייה הנכונה

### מציאת תיקיית הפלאגינים:
ב-LocalWP, לחצו על שם האתר → **Go to site folder** → פתחו את:
```
app → public → wp-content → plugins
```

### העתקת הפלאגין:
1. פתחו את תיקיית הפרויקט `SUNPEPE` (זו שמכילה את `sunpepe-landing.php`)
2. **העתיקו את כל תיקיית SUNPEPE** לתוך תיקיית `plugins`

התוצאה הסופית צריכה להיראות כך:
```
wp-content/
  plugins/
    sunpepe-landing/       ← שם התיקייה חייב להיות sunpepe-landing
      sunpepe-landing.php
      assets/
      includes/
      templates/
```

> **חשוב:** שם תיקיית הפלאגין חייב להיות `sunpepe-landing` (עם מקף, לא רווח).
> אם שם התיקייה שונה, שנו אותו לפני שממשיכים.

## 4. הפעלת הפלאגין

1. ב-WP Admin, לכו ל**פלאגינים** (Plugins) בתפריט הצד
2. מצאו ברשימה: **SUN PEPE Landing Page**
3. לחצו **Activate** (הפעל)
4. אם הצליח — הפלאגין יופיע ברשימה כ"פעיל" (Active) ותראו תפריט **SUN PEPE** בתפריט הצד השמאלי

> **אם הפלאגין לא מופיע** — ראו [פתרון תקלות](#פלאגין-לא-מופיע-ברשימה) בסוף המדריך.

## 5. יצירת עמוד חדש

1. ב-WP Admin, לכו ל**עמודים → הוסף חדש** (Pages → Add New)
2. בשדה "הכנס כותרת כאן", כתבו שם כלשהו, לדוגמה: `דף הבית`
3. **לפני שלוחצים "פרסם"** — המשיכו לשלב הבא

## 6. בחירת תבנית SUN PEPE Landing Page

1. בעמוד עריכת הדף (מהשלב הקודם), הסתכלו על **העמודה הימנית**
2. לחצו על **Page** (או **עמוד**) בחלונית הצד
3. מצאו את שדה **Template** (תבנית)
4. מהרשימה הנפתחת, בחרו: **SUN PEPE Landing Page**
5. לחצו **Publish** (פרסם)

> **אם "SUN PEPE Landing Page" לא מופיעה ברשימה** — ראו [פתרון תקלות](#התבנית-לא-מופיעה-ברשימת-התבניות) בסוף המדריך.

### הגדרה כדף הבית (אופציונלי):
1. לכו ל**הגדרות → קריאה** (Settings → Reading)
2. תחת **Your homepage displays**, בחרו **A static page**
3. תחת **Homepage**, בחרו את הדף שיצרתם
4. לחצו **Save Changes**

## 9. הוספת פריטי תפריט

> **לפני שמוסיפים פריטים:** התקינו ACF Free — ראו [שלב 10](#10-הגדרת-מחירים).
> ללא ACF, ניתן להוסיף פריטים אבל המחיר יוצג תמיד כ-₪0.

### הוספת פריט:

1. ב-WP Admin, לכו ל**SUN PEPE → פריטי תפריט**
2. לחצו **Add New** (הוסף חדש)
3. **כותרת** — שם הפריט, לדוגמה: `פיצה מוצרלה`
4. **קטגוריה** (בעמודה הימנית) — בחרו מהרשימה:
   - פיצות | סלטים | מאפים | פסטות | שתייה | תוספות
5. גללו מטה ל**פרטי פריט תפריט** (אם ACF מותקן):
   - **מחיר** — כתבו את המחיר בשקלים (ללא ₪)
   - **תיאור קצר** — אופציונלי, תיאור חד-משפטי
6. לחצו **Publish** (פרסם)
7. רענן את הדף הציבורי — הפריט יופיע בסעיף התפריט

### שינוי סדר פריטים:

1. פתחו עריכת הפריט
2. בעמודה הימנית, תחת **Page Attributes** → **Order** (סדר)
3. כתבו מספר — מספר נמוך יותר = מופיע קודם

### הסרת פריט:

- פתחו עריכת הפריט → לחצו **Move to Trash** (העבר לפח)
- פריט שנמצא בפח לא יופיע בדף

### קטגוריות:

הקטגוריות נוצרו אוטומטית בהפעלה ראשונה:
- פיצות | סלטים | מאפים | פסטות | שתייה | תוספות

לניהול קטגוריות: **SUN PEPE → פריטי תפריט → קטגוריות תפריט** (Menu Categories)

> **אם לא הוספתם פריטים עדיין** — הדף מציג תפריט דוגמה עם ₪0.
> ברגע שפריט אחד לפחות פורסם, הדף מציג את הפריטים האמיתיים בלבד.

## 10. הגדרת מחירים

המחירים נשמרים דרך פלאגין **ACF Free** (Advanced Custom Fields).

### התקנת ACF:

1. ב-WP Admin, לכו ל**פלאגינים → הוסף חדש**
2. בחיפוש כתבו: `Advanced Custom Fields`
3. מצאו את הגרסה של **WP Engine** (כיתוב: "Secure Custom Fields" או "Advanced Custom Fields")
4. לחצו **Install Now** → **Activate**

### עריכת מחיר:

1. לכו ל**SUN PEPE → פריטי תפריט**
2. לחצו על שם הפריט לעריכה
3. גללו מטה — תראו קופסה **"פרטי פריט תפריט"** עם שדה **מחיר (₪)**
4. הכניסו את המחיר (מספר בלבד, ללא סימן ₪)
5. לחצו **Update** (עדכן)

> **שדה המחיר לא מופיע?** בדקו ש-ACF מותקן ופעיל. ראו [פתרון תקלות](#שדה-מחיר-לא-מופיע).

## 12. פתרון תקלות נפוצות

---

### פלאגין לא מופיע ברשימה

**תסמין:** הפלאגין אינו מופיע ב-Plugins → Installed Plugins

**בדקו:**
1. האם שם תיקיית הפלאגין הוא בדיוק `sunpepe-landing` (לא `SUNPEPE`, לא `sunpepe-landing-main`)?
2. האם הקובץ `sunpepe-landing.php` נמצא ישירות בתוך התיקייה?
   - ✅ נכון: `plugins/sunpepe-landing/sunpepe-landing.php`
   - ❌ שגוי: `plugins/sunpepe-landing/SUNPEPE/sunpepe-landing.php`
3. האם יש שגיאת PHP? לכו ל-Tools → Site Health → Info ובדקו שגיאות

---

### התבנית לא מופיעה ברשימת התבניות

**תסמין:** בעמוד העריכה, בשדה Template, אין אפשרות "SUN PEPE Landing Page"

**בדקו:**
1. האם הפלאגין **פעיל** (Active)? לכו ל-Plugins ובדקו
2. האם הפלאגין אינו מציג שגיאה אדומה בלוח הבקרה?
3. נסו לבטל הפעלה ולהפעיל מחדש (Deactivate → Activate)
4. ודאו שגרסת WordPress היא 6.x ומעלה (Settings → General → WordPress Version)

---

### CSS לא נטען / הדף נראה שבור

**תסמין:** הדף מציג טקסט ללא עיצוב, צבעים לבנים, ללא פונטים

**בדקו:**
1. האם בחרתם בתבנית הנכונה בעמוד? (לא ברירת המחדל)
2. נסו Ctrl+F5 (hard refresh)
3. פתחו DevTools → Console — האם יש שגיאה אדומה? שמרו את הטקסט ובדקו בגוגל
4. ב-LocalWP, לכו ל-Site Settings → PHP Version — ודאו שגרסת PHP היא 8.1+
5. בדקו שאין תוסף cache שמגיש גרסה ישנה (ב-LocalWP בדרך כלל אין cache)

---

### פריטי תפריט לא מוצגים

**תסמין:** קטע התפריט בדף מציג תפריט דוגמה גם לאחר שהוספתם פריטים

**בדקו:**
1. האם הפריטים **פורסמו**? (Status = Published, לא Draft)
2. האם הפריטים שייכים ל**קטגוריה** (חייבים להיות עם קטגוריה)
3. רענן את הדף הציבורי (Ctrl+F5)
4. לכו ל-SUN PEPE → פריטי תפריט — האם הפריטים מופיעים ברשימה?
5. ב-All Posts ודאו שמצב הפוסט הוא "Published" (לא "Private" או "Pending")

---

### שדה מחיר לא מופיע

**תסמין:** בעמוד עריכת פריט תפריט, אין קופסה עם שדה "מחיר"

**בדקו:**
1. האם **ACF מותקן ופעיל**? לכו ל-Plugins → Advanced Custom Fields → Active ✓
2. ייתכן ש-Screen Options מסתיר את הקופסה — לחצו **Screen Options** (בפינה הימנית העליונה בעמוד העריכה) וודאו שמסומן **"פרטי פריט תפריט"**
3. נסו לבטל הפעלה ולהפעיל מחדש את ACF

---

### ACF מותקן אך מחיר לא מתעדכן בדף

**תסמין:** שיניתם מחיר ב-ACF אבל הדף עדיין מציג ₪0

**בדקו:**
1. לחצו **Update** (עדכן) בעמוד עריכת הפריט — לא רק שמרתם טיוטה
2. רענן את הדף הציבורי עם Ctrl+F5
3. ב-LocalWP, בדקו שאין cache פעיל (בדרך כלל אין)
4. ודאו שהפריט שייך לקטגוריה (ללא קטגוריה הפריט לא מוצג בדף כלל)

---

### הדף מציג מסך לבן / ריק

**תסמין:** הדף נפתח אבל לבן לגמרי (White Screen of Death)

**בדקו:**
1. ב-LocalWP, לכו ל-Logs (לוגים) ובדקו שגיאות PHP
2. ב-WP Admin, לכו ל-Tools → Site Health → Info → ובדקו WordPress Health
3. נסו להשבית את הפלאגין ולהפעיל מחדש
4. ודאו ש-PHP 8.1+ מותקן (Site Settings ב-LocalWP)
5. ודאו שגרסת WordPress היא 6.x ומעלה

**פתרון נפוץ:** גשו ל-`wp-content/plugins` ושנו את שם תיקיית `sunpepe-landing` ל-`sunpepe-landing-bak`. הרענן. אם הדף עכשיו לבן ופשוט — הבעיה בפלאגין. שחזרו את השם ובדקו שגיאות PHP בלוגים.

---

### כפתור ה-Sticky לא מוצג במובייל

**תסמין:** הכפתור הכחול/אדום בתחתית לא מופיע בטלפון

**בדקו:**
1. האם רוחב הדפדפן הוא מתחת ל-768px? (הכפתור מוצג רק ב-mobile)
2. בדקו ב-Chrome DevTools → Device Toolbar → בחרו מכשיר ספציפי
3. ייתכן שתוסף גורם להסתרה — בדקו ב-JS Console אם יש שגיאה

## מגבלות עיצוב (לצוות הפיתוח)

- עברית / RTL בלבד — אין תמיכה ב-LTR
- ללא טפסים, ללא analytics, ללא tracking, ללא מסרי משלוחים

- `ScrollTrigger.killAll()` **לא** נקרא — רק instances של SUN PEPE מנוקים
